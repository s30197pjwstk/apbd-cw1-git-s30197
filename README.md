Odpowiedzi:

1. Kiedy fast-forward, a kiedy merge commit?
Fast-forward - gdy main nie ma nowych commitow od momentu utworzenia galezi. 
Merge commit - gdy obie galeze maja nowe commity i Git musi je polaczyc.

2.  Czym w praktyce różni się merge od rebase?
Merge tworzy dodatkowy commit i historia nie jest liniowa.
Rebase przenosi commity na koniec galezi docelowej.

3. W jaki sposób został rozwiązany konflikt w Twoim repozytorium?
Konflikt byl w Program.cs - dwie rozne wersje komunikatu.
Wybralem nowa wersje laczaca oba warianty.
